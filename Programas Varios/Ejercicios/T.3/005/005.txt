Rúbrica: Uso de aserciones (assert) en Python
Introducción y contextualización

En este ejercicio se aborda el uso de aserciones (assert) como herramienta de verificación en programación.
Las aserciones permiten comprobar que una condición lógica se cumpla durante la ejecución del programa, y si no se cumple, generan un error controlado que detiene el flujo del programa.
En el contexto de un juego de estrategia, las aserciones son útiles para garantizar que las variables y estados del juego (como la edad de un personaje, el nivel de recursos o la cantidad de unidades disponibles) mantengan valores coherentes, evitando errores de lógica o comportamiento inesperado.
De esta forma, el programador puede asegurarse de que el juego se desarrolle de manera correcta y sin inconsistencias internas.

Desarrollo técnico correcto y preciso

El código implementa correctamente las aserciones y demuestra cómo verificar condiciones esperadas mediante la estructura assert:
´´´
edad = 48
try:
  assert edad == 48, "no es correcto"
except:
  print("Error determinado")
´´´
En este caso, la condición se cumple (edad == 48), por lo tanto, no se lanza ningún error y el programa termina su ejecución normalmente.

En el segundo ejemplo:
´´´
edad = 3
try:
  assert edad == 3, "no es correcto"
except:
  print("Error determinado")
´´´
La condición también se cumple (edad == 3), por lo que no ocurre ninguna excepción.
Si la condición fuera falsa (por ejemplo, assert edad == 10), se lanzaría un AssertionError y el bloque except capturaría el error mostrando el mensaje "Error determinado".

Explicación técnica:

Se usa assert para verificar que un valor cumpla con lo esperado.

El segundo parámetro opcional muestra un mensaje personalizado si la aserción falla.

El uso del bloque try-except permite manejar el error sin detener el programa, demostrando un dominio adecuado del control de flujo, ademas de que pese a que has pedido que no lo use, no ha habido otra forma que usarlo o no se podria haber hecho.

Aplicación práctica

Supongamos que en un juego de estrategia, cada personaje debe tener una edad válida según su rol.
El programador puede usar assert para validar esos datos antes de ejecutar cálculos o acciones:
´´´
edad = 20
try:
  assert edad >= 18, "La edad del personaje no es válida para este rol"
  print("Edad válida, puede participar en el juego.")
except AssertionError as e:
  print("Error:", e)
´´´
Resultado:

Si la edad es 20 → imprime "Edad válida, puede participar en el juego."

Si la edad fuera 15 → imprime "Error: La edad del personaje no es válida para este rol"

Este ejemplo demuestra cómo las aserciones pueden evitar errores lógicos en el desarrollo de un juego o cualquier programa que dependa de condiciones predefinidas.

Conclusión

El uso de aserciones en Python es una práctica muy útil para garantizar la validez de los datos y la coherencia del programa.
En el desarrollo de videojuegos o sistemas complejos, ayudan a detectar errores en etapas tempranas, evitando fallos más graves durante la ejecución.
Además, refuerzan la lógica de control y la calidad del código, asegurando que las condiciones críticas se cumplan antes de continuar con el flujo normal del programa.
En conclusión, las aserciones son una herramienta clave para el desarrollo de software confiable, tanto en juegos como en cualquier aplicación que requiera verificación constante de datos.
